1.Что такое структуры данных?
2.зачем они нужны?
3.что такое Hash Table?
4.что такое Linked List?
5.в каких случаях использовать Linked List и Hash Table?

1.Это способы хранить и организовывать данные.
2.Чтобы эффективней решать различные задачи.
3.Это структура данных, которая позволяет гораздо быстрее находить нужные данные.
4.Linked list это структура данных, которая для меньшего употребления процессов и памяти сохраняет не сам файл, а только
ссылку на него. Например Blockchain Nft который если бы сохранял каждую картинку каждого пользователя, то давно бы не
работал, вместо чего он сохраняет ссылки на картинки которые очень сильно экономят память.
5.Linked List нужно использовать когда хранится слишком много данных и не хватает памяти.А Hash Table когда среди данных
нужно найти определенный обьект.
